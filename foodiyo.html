Foodiyo Project

Developing a Recipe App

Overview

The goal of this project was to create an app to support people in finding recipe inspirations with ingredients they already have at home to prepare great dishes. The App was created during the Interaction Prototyping course of the Technical University of Munich.

Role

UI / UX Designer

Time Frame

6 Months

Skills

Figma
HMTL / CSS / JavaScript
Design thinking methods

Research

User Research

The most important user needs were determined by conducting user interviews. A lot of people are worried about not having all the ingredients at home for certain dishes and they are often struggle in finding recipes with the ingredients they got in their pantry.


Prototyping

Concept

The prototypes were developed for the Roboy Project at the Technical University in Munich, with the goal of designing and building an anthrophomorphic hand that can be adapted on the humanoid robot Roboy 2.0. This project is aimed to develop prototype solutions for grasping and manipulation operations. The hand is composed by a palm and five independent fingers, each with 22 degrees of freedom (hereinafter, DoF), actuated by servomotors and tendons.

Sketches

Started with sketches to come up with a proper user flow. User test were conducted by using the "Thinking-aloud"-method.

Low-fidelity Prototype

Created a low-fidelity prototype by using Figma with the main function "add ingredients to the digital pantry".

UML Diagram

The reqirements and high level functions of the applications are depicted in the UML use-case diagram.

Design Principles

Determined the most important design principles for the app.

Style Guide

Implemented customized design language based on user feedback.

A/B Testing

Conducted an A/B testing to decide on different color schemes and design approaches.

Quantitative Evaluation

Detemined the System Usability Scale (SUS), number of errors, the completion rate and the average task level satisfaction during the user tests.
